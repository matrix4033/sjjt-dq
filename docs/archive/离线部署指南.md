# 离线部署指南

## 适用场景

- 目标服务器无法连接外网
- 内网环境部署
- 安全隔离环境

---

## 部署流程

### 阶段 1：在有网络的机器上准备（开发机/跳板机）

#### 1. 下载依赖包

```bash
# 给脚本添加执行权限
chmod +x download-dependencies.sh

# 运行下载脚本
./download-dependencies.sh
```

这会创建：
- `offline-packages/` - 包含所有 Python 依赖包（.whl 文件）
- `offline-deployment.tar.gz` - 完整的离线部署包

#### 2. 传输到目标服务器

```bash
# 方式 1: 使用 scp
scp offline-deployment.tar.gz user@target-server:/path/to/destination/

# 方式 2: 使用 U 盘或其他介质
# 直接复制 offline-deployment.tar.gz 文件
```

---

### 阶段 2：在目标服务器上安装（无需网络）

#### 1. 解压部署包

```bash
# 解压
tar -xzf offline-deployment.tar.gz

# 进入目录
cd data-center  # 或你解压后的目录名
```

#### 2. 运行离线安装脚本

```bash
# 给脚本添加执行权限
chmod +x install-offline.sh

# 运行安装
./install-offline.sh
```

#### 3. 配置数据库连接

```bash
# 复制配置文件
cp api/.env.example api/.env

# 编辑配置
nano api/.env
```

修改为：
```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=data_center
```

#### 4. 初始化数据库

```bash
# 启动 MySQL（如果未启动）
sudo service mysql start

# 创建数据库和表
mysql -u root -p < setup-database.sql
```

#### 5. 导入数据

```bash
# 使用虚拟环境中的 Python
api/venv/bin/python import-csv-to-mysql.py
```

#### 6. 启动服务

```bash
# 启动所有服务
./start-all.sh

# 或分别启动
./start-backend.sh  # 后端
./start-frontend.sh # 前端
```

#### 7. 访问系统

```
http://localhost:8000
```

## 依赖包说明

### Python 依赖（从 requirements.txt）

```
Flask==3.0.0
pymysql==1.1.0
python-dotenv==1.0.0
Flask-Cors==4.0.0
```

### 系统依赖（需要预装）

目标服务器需要预先安装：
- Python 3.7+
- python3-venv
- python3-pip
- MySQL 8.0+

如果目标服务器没有这些，需要：

#### Ubuntu/Debian:
```bash
sudo apt install python3 python3-venv python3-pip mysql-server
```

#### CentOS/RHEL:
```bash
sudo yum install python3 python3-pip mysql-server
```

## 手动安装依赖（如果脚本失败）

### 1. 创建虚拟环境

```bash
python3 -m venv api/venv
```

### 2. 激活虚拟环境

```bash
. api/venv/bin/activate
```

### 3. 从离线包安装

```bash
# 安装单个包
pip install --no-index --find-links=offline-packages/ Flask

# 安装所有依赖
pip install --no-index --find-links=offline-packages/ -r api/requirements.txt
```

### 4. 验证安装

```bash
pip list
python -c "import flask, pymysql, dotenv, flask_cors"
```

### 5. 退出虚拟环境

```bash
deactivate
```

## 常见问题

### 1. 虚拟环境创建失败

**错误**: `The virtual environment was not created successfully`

**解决**:
```bash
# 安装 python3-venv
sudo apt install python3-venv
```

### 2. 依赖安装失败

**错误**: `Could not find a version that satisfies the requirement`

**原因**: 离线包不完整或 Python 版本不匹配

**解决**:
```bash
# 在有网络的机器上，指定 Python 版本下载
pip download -r api/requirements.txt -d offline-packages/ --python-version 3.8
```

### 3. MySQL 连接失败

**错误**: `Can't connect to MySQL server`

**解决**:
```bash
# 启动 MySQL
sudo service mysql start

# 检查状态
sudo service mysql status

# 测试连接
mysql -u root -p -e "SELECT 1;"
```

### 4. 权限问题

**错误**: `Permission denied`

**解决**:
```bash
# 给脚本添加执行权限
chmod +x *.sh

# 修改文件所有者
sudo chown -R $USER:$USER .
```

## 更新依赖包

如果需要更新依赖包：

### 1. 在有网络的机器上

```bash
# 清空旧包
rm -rf offline-packages/

# 重新下载
./download-dependencies.sh
```

### 2. 传输到目标服务器

```bash
# 只传输 offline-packages 目录
tar -czf offline-packages.tar.gz offline-packages/
scp offline-packages.tar.gz user@target-server:/path/to/project/
```

### 3. 在目标服务器上

```bash
# 解压
tar -xzf offline-packages.tar.gz

# 重新安装
. api/venv/bin/activate
pip install --no-index --find-links=offline-packages/ --upgrade -r api/requirements.txt
deactivate
```
