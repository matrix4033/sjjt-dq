# CSV 数据导入说明

## 使用方法

### 1. 确保数据库已创建

```bash
# 创建数据库和表结构
psql -U postgres -f setup-database-postgresql.sql
```

### 2. 准备 CSV 文件

确保以下文件存在：
- `data/汇总.csv` - 汇总数据
- `data/明细.csv` - 明细数据

### 3. 配置数据库连接

编辑 `api/.env` 文件，确保数据库配置正确：

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=sjjt_taskmonitor
DB_USER=postgres
DB_PASSWORD=your_password
```

### 4. 运行导入脚本

```bash
# 激活虚拟环境
source api/venv/bin/activate

# 运行导入脚本
python import-csv-data.py

# 或者直接使用虚拟环境的 Python
api/venv/bin/python import-csv-data.py
```

## CSV 文件格式

### 汇总.csv 格式

```csv
部门名称,部门表数量,前置机名称,前置机表数量,前置仓名称,前置仓表数量,贴源层名称,贴源层表数量
交通运输厅,82,149,82,前置仓,82,贴源层,82
```

### 明细.csv 格式

```csv
部门,标准文名,标英文名,部门到前置机状态,异常原因说明,前置机到前置仓状态,异常原因说明1,前置仓到贴源层状态,异常原因说明2
```

## 注意事项

1. 导入前会清空现有数据（TRUNCATE）
2. CSV 文件需要使用 UTF-8 编码
3. 确保数据库用户有相应的权限
4. 导入失败的行会显示警告，但不会中断整个导入过程

## 故障排查

### 连接失败

检查：
- PostgreSQL 服务是否运行
- 数据库配置是否正确
- 防火墙设置

### 文件不存在

确保 CSV 文件路径正确：
- `data/汇总.csv`
- `data/明细.csv`

### 编码问题

如果出现乱码，确保 CSV 文件使用 UTF-8 编码保存。
