## 拼音规则

字段名采用中文首字母拼音命名：
- 部门名称 → **b**u**m**en**m**ing**c**heng → bmmc
- 部门表数量 → **b**u**m**en**b**iao**s**hu**l**iang → bmbsl
- 前置机名称 → **q**ian**z**hi**j**i**m**ing**c**heng → qzjmc
- 创建时间 → **c**huang**j**ian**s**hi**j**ian → cjsj
- 更新时间 → **g**eng**x**in**s**hi**j**ian → gxsj

## API 返回数据格式

虽然数据库使用拼音字段名，但 API 返回给前端的数据会自动转换为中文字段名，保持前端代码不变。

### 示例：

**数据库查询**:
```sql
SELECT bmmc, bmbsl FROM hz;
```

**API 返回**:
```json
{
  "部门名称": "交通运输厅",
  "部门表数量": 82
}
```

这样前端 JavaScript 代码无需修改，仍然使用中文字段名访问数据。

## 迁移步骤

### 1. 删除旧表并创建新表

```bash
mysql -u root -p < setup-database.sql
```

### 2. 导入数据

```bash
. api/venv/bin/activate
python import-csv-to-mysql.py
deactivate
```

### 3. 重启服务

```bash
./start-all.sh
```

## 注意事项

1. **前端无需修改**：API 层已做字段名转换，前端仍使用中文字段名
2. **数据库规范**：数据库层使用拼音字段名，符合命名规范
3. **向后兼容**：通过 SQL 别名实现兼容性
4. **性能优化**：拼音字段名更短，提高查询效率
