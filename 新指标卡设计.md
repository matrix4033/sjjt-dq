# 指标卡重新设计方案

## 指标定义

### 1. 汇聚部门数
- **数据源**: 汇总.csv
- **计算方式**: 【部门名称】的去重数量
- **单位**: 个
- **图标**: 🏢
- **颜色**: #00D9FF (蓝色)

### 2. 部门汇聚清单表数量
- **数据源**: 汇总.csv
- **计算方式**: 【部门表数量】的总和
- **单位**: 张
- **图标**: 📊
- **颜色**: #00FF88 (绿色)

### 3. 部门数据更新率
- **数据源**: 明细.csv
- **计算方式**: 【部门到前置机状态】为"正常"的数量 / 总数量 × 100%
- **单位**: %
- **图标**: ⬆
- **颜色**: #FF9500 (橙色)

### 4. 前置仓更新率
- **数据源**: 明细.csv
- **计算方式**: 【前置机到前置仓状态】为"正常"的数量 / 总数量 × 100%
- **单位**: %
- **图标**: 📦
- **颜色**: #AA00FF (紫色)

### 5. 链路完成率
- **数据源**: 明细.csv
- **计算方式**: 【部门到前置机状态】、【前置机到前置仓状态】、【前置仓到贴源层状态】都为"正常"的数量 / 总数量 × 100%
- **单位**: %
- **图标**: ✓
- **颜色**: #00CC66 (深绿色)

### 6. 链路异常率
- **数据源**: 明细.csv
- **计算方式**: 【部门到前置机状态】、【前置机到前置仓状态】、【前置仓到贴源层状态】任意一个为"异常"的数量 / 总数量 × 100%
- **单位**: %
- **图标**: ⚠
- **颜色**: #FF4444 (红色)

## 布局设计

采用第一行2列（较宽）+ 第二行4列（较窄）的网格布局：

```
┌─────────────────────────┬─────────────────────────┐
│     汇聚部门数          │  部门汇聚清单表数量     │
│      ● XX个             │      ● XXX张            │
└─────────────────────────┴─────────────────────────┘
┌───────────┬───────────┬───────────┬───────────┐
│部门更新率 │前置仓更新率│链路完成率 │链路异常率 │
│ ● XX.X%   │ ● XX.X%   │ ● XX.X%   │ ● XX.X%   │
└───────────┴───────────┴───────────┴───────────┘
```

## 样式特点
- 暗黑科技风格
- 使用发光圆点（●）代替图标
- 每个指标卡有对应颜色的发光效果
- 第一行2个卡片较宽，显示基础统计
- 第二行4个卡片较窄，显示更新率指标
- 数值大字体显示
- 标签小字体显示
- 响应式布局
