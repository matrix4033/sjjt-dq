## 📦 部署步骤概览

### 1. 安装 PostgreSQL

```bash
# Ubuntu
sudo apt install -y postgresql postgresql-contrib

# CentOS
sudo yum install -y postgresql12-server postgresql12

# Windows
# 下载安装器: https://www.postgresql.org/download/windows/
```

### 2. 创建数据库

```bash
sudo -u postgres psql
CREATE DATABASE data_center;
\q
```

### 3. 安装 Python 依赖

```bash
python3 -m venv api/venv
source api/venv/bin/activate  # Windows: api\venv\Scripts\activate.bat
pip install -r api/requirements.txt
deactivate
```

### 4. 配置应用

```bash
cp api/.env.example api/.env
# 编辑 api/.env 配置数据库连接
```

### 5. 初始化数据

```bash
psql -U postgres -d data_center -f setup-database-postgresql.sql
python3 import-csv-to-postgresql.py
```

### 6. 启动服务

```bash
./start-all.sh  # Linux
start-all.bat   # Windows
```

## 🔒 生产环境建议

### 1. 使用系统服务

```bash
sudo ./install-systemd-service.sh
sudo systemctl enable data-center-api
sudo systemctl enable data-center-frontend
```

### 2. 配置 Nginx 反向代理

```bash
sudo cp nginx/data-center.conf /etc/nginx/sites-available/
sudo ln -s /etc/nginx/sites-available/data-center.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 3. 启用 HTTPS

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 4. 配置防火墙

```bash
sudo ufw enable
sudo ufw allow ssh
sudo ufw allow 'Nginx Full'
```

### 5. 设置定期备份

```bash
# 创建备份脚本
cat > /opt/data-center/backup.sh <<'EOF'
#!/bin/bash
BACKUP_DIR="/opt/data-center/backups"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
pg_dump -U postgres data_center | gzip > $BACKUP_DIR/data_center_$DATE.sql.gz
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete
EOF

chmod +x /opt/data-center/backup.sh

# 添加到 crontab（每天凌晨 2 点）
(crontab -l 2>/dev/null; echo "0 2 * * * /opt/data-center/backup.sh") | crontab -
```

## 🐛 常见问题

### Q1: 数据库连接失败

**检查清单**:
1. PostgreSQL 服务是否运行
2. 配置文件 `api/.env` 是否正确
3. 数据库是否已创建
4. 用户权限是否足够

**解决**:
```bash
sudo systemctl status postgresql
cat api/.env
psql -U postgres -d data_center
```

### Q2: Python 依赖安装失败

**常见原因**:
- 缺少开发工具
- 网络问题
- Python 版本不兼容

**解决**:
```bash
# Ubuntu
sudo apt install -y python3-dev libpq-dev build-essential

# CentOS
sudo yum install -y python3-devel postgresql-devel gcc
```

### Q3: 端口被占用

**解决**:
```bash
# 查看占用端口的进程
sudo lsof -i :5000
sudo lsof -i :8000

# 结束进程
sudo kill -9 PID

# 或使用停止脚本
./stop-all.sh
```

### Q4: 前端无法访问 API

**检查清单**:
1. 后端是否正常运行
2. CORS 配置是否正确
3. 防火墙是否阻止
4. API 地址是否正确

**解决**:
```bash
# 测试 API
curl http://localhost:5000/api/health

# 查看后端日志
tail -f backend.log
```

## 📊 性能优化

### 1. 使用 Gunicorn (生产环境)

```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 api.app:app
```

### 2. PostgreSQL 优化

编辑 `/etc/postgresql/14/main/postgresql.conf`:

```conf
shared_buffers = 256MB
effective_cache_size = 1GB
work_mem = 16MB
maintenance_work_mem = 64MB
```

### 3. Nginx 缓存

在 Nginx 配置中添加缓存:

```nginx
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=api_cache:10m;

location /api/ {
    proxy_cache api_cache;
    proxy_cache_valid 200 5m;
    proxy_pass http://localhost:5000;
}
```
